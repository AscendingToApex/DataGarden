## Overview

This reference focuses exclusively on **data literacy concepts** - the foundational knowledge required to understand, interpret, and communicate data effectively. It is designed to build shared understanding across all roles, independent of specific tools or technical implementations.

---

## 1. What Is Data Literacy?

**Data literacy** is the ability to:

* Read data (interpret tables, charts, and metrics)
* Understand data (context, definitions, limitations)
* Analyze data (identify patterns and relationships)
* Communicate data (translate findings into clear insights)

### Why It Matters

* Enables better decision-making
* Reduces misinterpretation and bias
* Improves cross-team alignment

---

## 2. Core Concepts

* **Data**: Raw facts or observations
* **Information**: Data with context and meaning
* **Insight**: Actionable understanding derived from data
* **Metric**: A measurable value
* **KPI**: A metric tied to a business objective
* **Dimension**: A category used to group data (e.g., region, product)
* **Measure**: A quantitative value (e.g., revenue, count)
* **Bias**: Systematic error in data or interpretation
* **Signal vs Noise**: Meaningful patterns vs random variation

---

## 3. Analytical Thinking & Mental Models

### Ways of Thinking

* **Top-down**: Start with a question, break into components
* **Bottom-up**: Explore data to discover patterns
* **Hypothesis-driven**: Form assumptions and test them

### Key Mental Models

* **Segmentation**: Breaking data into meaningful groups
* **Cohort thinking**: Comparing groups over time
* **Funnel thinking**: Understanding step-by-step processes
* **Base rate awareness**: Considering overall context before conclusions
* **Causation vs Correlation**: Distinguishing relationships from causes

---

## 4. Data Types & Structures (Conceptual)

### Data Types

| Type        | Description         | Example         |
| ----------- | ------------------- | --------------- |
| Numerical   | Quantitative values | Revenue, age    |
| Categorical | Labels/groups       | Region, product |
| Temporal    | Time-based values   | Date, quarter   |
| Boolean     | True/false values   | is_active       |
| Text        | Free-form content   | Comments        |

### Structural Concepts

* **Structured data**: Organized in rows and columns
* **Semi-structured data**: Flexible formats (e.g., JSON)
* **Unstructured data**: No predefined format (e.g., text, images)

---

## 5. Data Context & Definitions

Understanding data requires context:

### Key Elements

* **Definitions**: What each metric means
* **Scope**: What is included/excluded
* **Timeframe**: When data was collected
* **Granularity**: Level of detail (daily, monthly, per user)

### Example

“Active users” may mean:

* Last 7 days
* Last 30 days
* Logged in vs performed an action

---

## 6. Data Quality Concepts

### Dimensions of Data Quality

* **Accuracy**: Reflects reality correctly
* **Completeness**: Required data is present
* **Consistency**: Uniform definitions and formats
* **Timeliness**: Data is up to date
* **Validity**: Values fall within expected ranges

### Why It Matters

Poor data quality leads to:

* Incorrect conclusions
* Loss of trust
* Poor decisions

---

## 7. Interpreting Data

### Descriptive Understanding

* Averages (mean, median)
* Spread (range, variability)

### Relationships

* Correlation (variables move together)
* Not necessarily causation

### Trends

* Changes over time
* Seasonality patterns

### Outliers

* Unusual values that require investigation

---

## 8. Common Analytical Patterns

* **Comparison**: This vs that (e.g., regions, time periods)
* **Trend analysis**: How something changes over time
* **Distribution analysis**: How values are spread
* **Segmentation**: Differences across groups

---

## 9. Data Visualization Literacy

### Interpreting Charts

* **Line charts**: Trends over time
* **Bar charts**: Comparisons across categories
* **Histograms**: Distributions
* **Scatter plots**: Relationships

### What to Look For

* Axes and labels
* Scale and units
* Data ranges
* Highlighted insights

### Common Misinterpretations

* Misleading axes
* Ignoring context
* Overemphasizing small differences

---

## 10. Communicating with Data

### Principles

* Be clear and concise
* Focus on the key message
* Provide context and definitions
* Highlight limitations

### Basic Structure

1. Question
2. Key finding
3. Supporting data
4. Implication

---

## 11. Common Pitfalls

* Confusing correlation with causation
* Ignoring sample bias
* Overgeneralizing from small samples
* Misunderstanding definitions
* Cherry-picking data
* Ignoring missing context

---

## 12. Data Ethics & Responsibility

### Principles

* Use data honestly
* Avoid misleading representations
* Respect privacy and confidentiality
* Acknowledge uncertainty

---

## 13. Glossary

* **KPI**: Key Performance Indicator
* **Dimension**: Category used for grouping
* **Measure**: Numeric value
* **Cohort**: Group sharing a common characteristic
* **Bias**: Systematic error

---

## Summary

Data literacy is about understanding what data represents, how it behaves, and how to communicate it responsibly. Strong data literacy enables better questions, clearer insights, and more informed decisions across the organization.
